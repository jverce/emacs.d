# Emacs Configuration

## Architecture

### Load Order

1. `early-init.el` -- Sets `gc-cons-threshold` to `most-positive-fixnum` to defer GC during startup
2. `init.el` -- Configures package archives (MELPA, MELPA Stable, GNU ELPA), enables `envrc`, installs `setup.el`, then loads each file in the `addons` list from `customizations/`
3. `custom.el` -- Auto-generated by Emacs customize system, loaded last. Gitignored; do not edit manually

### Package Management

Uses built-in `package.el`. Config macros: `use-package` (built-in since Emacs 29) and `setup.el` (installed from ELPA). Both are used throughout -- this dual pattern is intentional; there is no single convention.

### Environment

- `envrc` for direnv/nix integration (configured in `init.el`)
- `exec-path-from-shell` for macOS PATH (in `shell-integration.el`)

### Theme

GitHub Dark Dimmed via `autothemer`, loaded from git submodule at `themes/github/`.

### TreeSitter

`treesit-auto` for automatic grammar management (configured in `editing.el`). Explicit grammar sources for Go, Go modules, TSX, and TypeScript in `editing.el`. YAML is excluded from `treesit-auto-langs`.

## File Organization (customizations/)

The `addons` list in `init.el` loads these files in order:

| File | Purpose |
|---|---|
| `ui.el` | Frame settings, fonts (FiraCode Nerd Font), doom-modeline, all-the-icons, theme loading |
| `navigation.el` | which-key, ivy/counsel/swiper completion framework, ivy-rich |
| `projects.el` | Projectile, counsel-projectile, ag |
| `git.el` | Magit (pinned to melpa-stable) |
| `filetree.el` | Treemacs + treemacs-projectile + treemacs-magit, ace-window |
| `editing.el` | General editor behavior (hippie-expand, line numbers, paren matching, backups, company-mode, treesit-auto, evil-nerd-commenter) |
| `lsp.el` | LSP-mode config, keybindings (prefix `C-c l`), lsp-ui, lsp-ivy, lsp-treemacs, dap-mode, format-on-save |
| `go.el` | go-ts-mode with LSP, format + organize imports on save |
| `terraform.el` | terraform-mode with outline-minor-mode, format-on-save |
| `setup-python.el` | python-ts-mode remap, tree-sitter grammar auto-install, ruff format-on-save |
| `yaml.el` | yaml-mode with LSP |
| `elisp-editing.el` | Paredit, eldoc, rainbow-delimiters for Lisp modes |
| `setup-clojure.el` | Clojure toolchain: clojure-mode, CIDER, clj-refactor, cider-hydra, paredit |
| `setup-js.el` | JavaScript/HTML (tagedit, js-ts-mode with LSP, CoffeeScript) |
| `shell-integration.el` | `exec-path-from-shell` for macOS |

## Key Conventions

- All `.el` files use `lexical-binding: t`
- Language configs follow: TreeSitter mode remap + LSP hook + format-on-save
- `custom.el` is gitignored and auto-generated -- don't edit it manually
