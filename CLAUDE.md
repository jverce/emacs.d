# Emacs Configuration

## Architecture

### Load Order

1. `early-init.el` — Sets `gc-cons-threshold` to `most-positive-fixnum` to defer
   GC during startup.
2. `init.el` — Configures package archives (MELPA, MELPA Stable, GNU ELPA),
   bootstraps a small set of required packages (`flycheck`, `lsp-mode`,
   `ruff-format`), resolves the shell PATH, enables `envrc-global-mode`, then
   auto-loads every `.el` file in `customizations/` in alphabetic order.
3. `custom.el` — Auto-generated by Emacs's customize system, loaded after
   the customizations directory. Gitignored; do not edit manually.
4. `local.el` — Optional, gitignored, loaded last for per-machine overrides.

### Package Management

Uses built-in `package.el`. Configuration uses **`use-package` only**
(`setup.el` was removed during the cleanup pass). Package installation in
customization files is via `:ensure t` on the `use-package` form.

### Environment

- `envrc` for direnv / nix integration (configured in `init.el`).
- `init.el` resolves PATH from a deterministic login shell (`/bin/zsh` on
  macOS, `/bin/bash` on Linux) so daemon sessions started by `launchd` /
  `systemd` see the same PATH as a normal terminal.

### Theme

GitHub Dark Dimmed via `autothemer`, loaded from a git submodule at
`themes/github/`.

### TreeSitter

`treesit-auto` for automatic grammar management (configured in `15-editing.el`).
Explicit grammar sources for Go, Go modules, TSX, and TypeScript are also in
`15-editing.el`. YAML is intentionally excluded from `treesit-auto-langs`.

## File Organization (`customizations/`)

`init.el` auto-loads every `.el` under `customizations/` in alphabetic order.
Numeric prefixes on core modules and the `lang-` prefix on language modules
document and enforce load order — there is no hand-curated list.

## The `my/define-language` helper

`01-lib-language.el` defines `my/define-language`, a small declarative macro
for the common pattern of *tree-sitter mode + extensions + LSP + formatter*.
Use it for new language modules whose needs fit the pattern; write a
hand-rolled `lang-<name>.el` when they don't.

```elisp
(my/define-language rust
  :mode rust-ts-mode
  :extensions ("\\.rs\\'")
  :lsp t
  :formatter lsp-format-buffer)
```

Arguments:

- `:mode` — major mode symbol (required). Adds `lsp-deferred` /
  formatter / extra hooks to `<mode>-hook`.
- `:extensions` — list of regexps for `auto-mode-alist`.
- `:lsp` — non-nil to add `lsp-deferred` to `<mode>-hook`.
- `:formatter` — function symbol. Names ending in `-mode` are enabled as a
  buffer-local minor mode; everything else is added to `before-save-hook`
  buffer-locally.
- `:save-hooks` — list of functions added to `before-save-hook` (use this
  when you need several save-time actions).
- `:extra-hooks` — list of additional functions appended to `<mode>-hook`.

## Per-machine overrides (`local.el`)

`init.el` loads `~/.emacs.d/local.el` last when it exists. The file is
gitignored — use it for machine-specific font sizes, projectile search
paths, work-only LSP clients, and anything else that shouldn't live in the
repo.

## Key Conventions

- Every `.el` file uses `lexical-binding: t` and has `;;; <name>.el ---`
  / `;;; <name>.el ends here` headers and a `(provide '<name>)` form.
- Language modules follow this pattern, in this order:
  1. `(use-package <package> :ensure t ...)` for any package that needs
     installation.
  2. `my/define-language` call (or a hand-rolled equivalent).
  3. Any extra helpers specific to the language.
- `custom.el` and `local.el` are gitignored and not part of the source tree.
